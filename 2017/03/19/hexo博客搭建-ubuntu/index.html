hexo博客搭建-ubuntu

hexo是什么？

简言之，可以认为hexo是一个markdown –> html转换器。

0.准备

hexo的安装依赖于git和nodejs，因此安装hexo之前，需要安装git和nodejs，最后才安装hexo

  • 安装git

    sudo apt-get install git-core

  • 安装nodejs：用npm安装nodejs(ps: npm是nodejs的包管理工具)

    curl https://raw.githubusercontent.com/creationix/nvm/master/install.sh | sh

    nvm install stable

  • 安装hexo

    npm install -g hexo-cli

    重启终端后，即可使用hexo命令。

1. hexo

1.1 初始化博客

hexo初始化博客

以上就构建好了专属自己的博客，直接运行hexo server就能在本地4000端口查看到自己的博客，里面有一篇默认的hello world博文。

1.2 博客目录详解

详细地，在运行hexo init时，blog目录从一个空文件夹变成了具有如下结构的目录：

.
├── _config.yml
├── package.json
├── scaffolds
├── source
| ├── _drafts
| └── _posts
└── themes

文件/目录 说明
_config.yml 站点配置文件。绝大部分的配置选项都在这儿 详见：configuration
package.json 好像是哪些插件或渲染器要安装的配置？
scaffolds/ 博文的layout布局模板 draft、post、page三种模板。如hexo new draft hexo-博客搭建-ubuntu就能根据draft.md里的定义生成hexo博客搭建-ubuntu.md的草稿开始撰写。
source/ 博文的源文件目录 如：hexo博客搭建-ubuntu.md的草稿版本会在source/_draft/目录；如默认生成的post版本的hello world.md在source/_post/目录
themes/ 博客的主题、样式、风格
public/* html,css,js等通过(hexo generate)生成的静态文件 非初始化的子文件夹，以后执行hexo generate产生的

1.4 部署

前面提到执行hexo server即可在本地部署博客网站，通过localhost:4000访问，但若要能在外网访问，即部署到互联网上，那么就要借助github pages了。简言之，github pages既提供了域名(username.github.io)，也提供了主机(虚拟web服务器)，因此其他人可以通过域名访问该主机。而我们只需要把生成的静态文件同步到github的相应repository即可。

  1. 注册github，并新建repository。如github的帐号名为username，则新建的repository的名称就必须设为username.github.io。相应的个人博客网站的域名也就是username.github.io
  2. 在终端执行hexo generate，将所有的博文生成html等静态文件，存放在blog的public/目录下。
  3. hexo deploy部署
    1. 修改配置文件_config.yml定义部署的方式为git：url定义为username.github.io，deploy的type定义为git，新增repo定义为该git repository的地址，branch定义为master
    2. 安装hexo-deployer-git，以支持git部署。
    3. hexo deploy即可。若报错git还未设置email和name，需先设置好git config --global user.email "you@example.com"，git config --global user.name "Your Name"
  4. 浏览器打开username.github.io即可
