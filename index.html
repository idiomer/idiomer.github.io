推荐系统

前言

​ 推荐系统是为用户(user)推荐感兴趣的物品(item)的系统。其中感兴趣的量化指标在不同的场景有不同的定义：如电商/广告行业注重转化率，视频网站注重点击率的同时还要保持新鲜度等。

​ 一般而言，推荐系统的搭建，需要历史的user-item矩阵的评分(rating)。如用户给电影打4分，用户点击了某物品打2分，用户购买了某物品则打10分，用户收藏了某音乐则打8分等。

1. 基于用户投票的推荐系统

简单来说，就是对每个item的所有用户评分计算出平均值，然后给出top排行榜。其中，有如下几个注意点：

  • 有些item的用户评分比较少，有些非常多，因此，平均值的计算常常涉及到平滑的问题
  • 排行榜可以分维度给出，如按国家，按item类型等

2. 基于内容的推荐系统

简单来说，就是根据item属性(内容)来计算item之间的相似度矩阵。有了这个相似度矩阵后，再根据用户评分高的item来推荐相似的items。

  • 核心是如何找到强相关的内容，来计算item之间的相似度
  • 推荐了相似items后，可以根据上述的排行榜再排序

3. 协同过滤推荐

从字面理解，协同就是寻找共同点，过滤就是筛选出优质的内容。

3.1 NormalPredictor

fit：对所有的评分估计出一个正态分布（Normal Distribution，也称高斯分布）

predict：与user和item无关，按正态分布随机给出一个评分

分析：用这种方式构建的推荐系统，在评估指标上的loss看上去确实还算低。但是每个评分都是随机的，因此如果看testset的真实评分和随机预测的评分的相关系数corr，就可以发现corr接近于0，也就是说算法基本没有排序能力。

3.2 BaselineOnly

fit：

  1. 计算所有评分的平均值$\mu$
  2. 对每个用户u，估算出这个用户评分的严格程度$b_u$（比如这个用户的所有评分都偏低，也就是比较严格，所以$b_u$是个负数，如-0.3）
  3. 对每个物品i，估算出这个物品的质量$b_i$（比如这个物品的平均评分远远高于所有物品的平均评分，那么说明这个物品的质量好，$b_i$估算出来是个正数，如0.8）

predict：

  1. $b_{ui} = \mu + b_u + b_i$
  2. 通俗理解，用户u对物品i的评分是：首先是全局平均分$\mu$，然后这个用户习惯好评还是差评$\mu + b_u$，最好这个物品质量是否比较好$\mu + b_u + b_i$

分析：simple yet accurate！通俗易懂，想法朴素，但效果极好。然而，由公式可知，对于用户来说，$\mu + b_u$都是相同的，也就是说，给用户的推荐是由$b_i$决定的，而$b_i$是和用户无关的。因此对每个用户来说，其推荐结果都是一样的。因此，基本就是排行榜一样的效果

3.3 Slope One

fit：

  1. 遍历所有物品i，所有物品j，其中i!=j，找出所有同时评分过i和j的用户集合$U_{ij}$，记录用户数$N_{ij} = |U_{ij}|$
  2. 以及$U_{ij}$对应的评分$S_i$和$S_j$，计算$R_{ij} = mean(S_i - S_j)$。也就是，物品j的评分和物品i的评分的平均差别

predict：

  1. 遍历用户u的评分过的物品(排除i)，这些物品记为$j$
  2. 这个用户u对于物品i的预估评分是：
  3. $\hat{r}{ui} = \frac{\sum{j\in I_u } N_{ij} (r_{uj} + R_{ij})}{\sum_{j\in I_u } N_{ij}}$，其中$I_u$是用户u评分过的所有Items（排除物品$i$）

分析：对每个物品，都先估算一下和其他物品的平均偏差($R_{ij}$)；然后对每个用户，都用这个用户在其他物品上的评分，通过偏差修正的方式，来预估这个用户给当前物品的评分；最后是用每个偏差的支持度($N_{ij}$, support)做为加权和的权值。算法思想很朴素，容易理解。

3.3 基于近邻的协同过滤

KNNBasic

fit：直接保存user-item-rating矩阵

predict：

  1. 对于用户u和物品i，以user-based为例
  2. 选取K个相邻用户v
  3. 查询这些用户v在该物品i上的评分scores
  4. 计算这些用户v各自和用户u的相似度sims
  5. 然后以相似度为权重，加权得到预估评分$\hat{r}_{ui} = \frac{sims \cdot scores}{sum(sims)}$

分析：以上便是KNNBasic的计算流程，也是最朴素最基本的协同过滤。一句话理解就是：和””我””相似的其他人对这个物品的评分是多少，越相似的人在””我””这里的权重越高。

KNNBaseline改进

$$\hat{r}{ui} = b{ui} + \frac{sims \cdot (scores-b_{vi})}{sum(sims)}$$

KNNMeans改进

$$\hat{r}_{ui} = \mu_u + \frac{sims \cdot (scores - \mu_v)}{sum(sims)}$$

KNNZscore改进

$$\hat{r}_{ui} = \mu_u + \sigma_u \frac{sims \cdot \frac{(scores - \mu_v)}{\sigma_v}}{sum(sims)}$$

初步在jupyter上使用facets神器

背景

​ 昨天在逛知乎的时候，偶然发现了facets这个可视化的神器。然后去web版体验了一下功能，非常棒：overview能快速对自己的数据集有一个快速且全面的了解，进而能快速地发现其中的异常数据；dive的功能更是炫丽，通过图表+缩放的逆天前端实现，直观快速地了解多维度组合下的特征区分度。OK! Talk is cheat, show me the picture

​ 虽然web版可以直接使用，但是考虑到数据安全性的关系，特别是企业业务的关键数据，在自己的jupyter上搭建facets的可控性是最好的，并且可以灵活处理数据。

安装步骤

前提：安装好了jupyter

  1. 安装protobuf：pip3 install protobuf
  2. 安装facets：
    1. git clone https://github.com/PAIR-code/facets
    2. cd facets/
    3. jupyter nbextension install facets-dist/ --user
  3. 测试是否正常运行
    1. 开启jupyter后，运行facets/facets_overview/Overview_demo.ipynb这个notebook，看是否成功
    2. 看到能正常出现overview的界面即可（注意：当前只支持chrome浏览器）
    3. 之后按这个demo的notebook，换成自己的数据集即可
  4. 优化：将facest_overview/python/*.py复制到/usr/lib/python3.5/facets/（require mkdir）里面去
    1. 这样就不需要sys.argv.append(‘./python’)了
    2. notebook demo中的import语句，将from generic_feature_statistics_generator import GenericFeatureStatisticsGenerator 改成from facets.generic_feature_statistics_generator import GenericFeatureStatisticsGenerator

jupyter插件配置——为数据分析插上翅膀

背景

看到知乎上一篇高票回答，方知jupyter加上插件后就变成神器了，马上在自己的笔记本上跟进配置

步骤

  • 注意，因为jupyter已经不支持python 2.x了，用历史版本安装的话，装插件的时候会有一些问题，因此推荐python3.5+。（2.x的我流下了心酸的泪水55555）

  1. 安装ipython和jupyter（因为主要用来做数据分析，所以顺便安装了numpy，pandas等）

    • pip3 install –user numpy scipy matplotlib ipython jupyter pandas sympy nose

  2. 安装jupyter_contrib_nbextensions

    1. pip3 install jupyter_contrib_nbextensions # Install the python package
    2. jupyter contrib nbextension install –user # Install javascript and css files
    3. jupyter nbextension enable \ # Enabling/Disabling extensions
      • 或者在web界面上开启/关闭某些插件
      • 也就是开启jupyter后，访问http://localhost:8888/tree#nbextensions_configurator
    4. 开启一些建议的插件（点击插件后，web下方有readme介绍怎么用，非常nice）
      • Hinterland
      • Split Cells Notebook
      • Snippets
    5. 到此，已经有一些nbextensions的插件可以用了。但是不是所有好用的插件都在nbextensions里面，所以还可以装一些其他的”小工具”，如ipywidgets。

  3. 安装ipywidgets：略，个人感觉不是很需要

  4. 安装Qgrid：可以像excel一样筛选pandas的dataframe，非常实用

    1. pip3 install qgrid
    2. (sudo) jupyter nbextension enable –py –sys-prefix qgrid
    3. jupyter nbextension enable –py –sys-prefix widgetsnbextension

Hexo迁移

Hexo迁移

当换电脑或者多设备编辑时，需要部署两个或多个hexo，怎么保持同步呢？

  1. 在新设备上按hexo教程初始化一个hexo目录
    1. hexo init
    2. npm install
    3. npm install hexo-deployer-git –save
  2. 将旧项目的_config.yml 和 source/ 和scaffolds/ 和themes/ 替换新项目的相应位置即可
  3. 正常写作，并生成部署
    1. hexo new post “title”
    2. hexo g
    3. hexo deploy

hexo博客搭建-ubuntu

hexo是什么？

简言之，可以认为hexo是一个markdown –> html转换器。

0.准备

hexo的安装依赖于git和nodejs，因此安装hexo之前，需要安装git和nodejs，最后才安装hexo

  • 安装git

    sudo apt-get install git-core

  • 安装nodejs：用npm安装nodejs(ps: npm是nodejs的包管理工具)

    curl https://raw.githubusercontent.com/creationix/nvm/master/install.sh | sh

    nvm install stable

  • 安装hexo

    npm install -g hexo-cli

    重启终端后，即可使用hexo命令。

1. hexo

1.1 初始化博客

hexo初始化博客

以上就构建好了专属自己的博客，直接运行hexo server就能在本地4000端口查看到自己的博客，里面有一篇默认的hello world博文。

1.2 博客目录详解

详细地，在运行hexo init时，blog目录从一个空文件夹变成了具有如下结构的目录：

.
├── _config.yml
├── package.json
├── scaffolds
├── source
| ├── _drafts
| └── _posts
└── themes

文件/目录 说明
_config.yml 站点配置文件。绝大部分的配置选项都在这儿 详见：configuration
package.json 好像是哪些插件或渲染器要安装的配置？
scaffolds/ 博文的layout布局模板 draft、post、page三种模板。如hexo new draft hexo-博客搭建-ubuntu就能根据draft.md里的定义生成hexo博客搭建-ubuntu.md的草稿开始撰写。
source/ 博文的源文件目录 如：hexo博客搭建-ubuntu.md的草稿版本会在source/_draft/目录；如默认生成的post版本的hello world.md在source/_post/目录
themes/ 博客的主题、样式、风格
public/* html,css,js等通过(hexo generate)生成的静态文件 非初始化的子文件夹，以后执行hexo generate产生的

1.3 新增博文

1.4 部署

前面提到执行hexo server即可在本地部署博客网站，通过localhost:4000访问，但若要能在外网访问，即部署到互联网上，那么就要借助github pages了。简言之，github pages既提供了域名(username.github.io)，也提供了主机(虚拟web服务器)，因此其他人可以通过域名访问该主机。而我们只需要把生成的静态文件同步到github的相应repository即可。

  1. 注册github，并新建repository。如github的帐号名为username，则新建的repository的名称就必须设为username.github.io。相应的个人博客网站的域名也就是username.github.io
  2. 在终端执行hexo generate，将所有的博文生成html等静态文件，存放在blog的public/目录下。
  3. hexo deploy部署
    1. 修改配置文件_config.yml定义部署的方式为git：url定义为username.github.io，deploy的type定义为git，新增repo定义为该git repository的地址，branch定义为master
    2. 安装hexo-deployer-git，以支持git部署。
    3. hexo deploy即可。若报错git还未设置email和name，需先设置好git config --global user.email "you@example.com"，git config --global user.name "Your Name"
  4. 浏览器打开username.github.io即可

What is github pages?

github pages相当于一个虚拟web服务器主机，每个github帐号只能有一个github page。如果你的github帐号是abc，那么你的page就会是abc.github.io。那么怎么开始呢？

  1. register a github：先注册一个github帐号，如帐号名为abc

  1. create a repository：创建一个新的项目仓库，命名为\<username>.github.io，即abc.github.io。

  2. clone the repository：克隆一份到本地

    git clone https://github.com/username/username.github.io

  3. hello world：创建你的主页html文件，如

    cd username.github.io

    echo “Hello world! Welcome to my page!” > index.html

  4. push it：推送本地仓库到github同步

  5. done：用浏览器打开username.github.io。Welcome to my page!
